# SNAPFIRE 答辩问答预案

> 叶晨宇 负责部分：OBJ 序列动画 · miniaudio 音频 · 答辩

---

## Q1：开场 OBJ 序列动画怎么实现的？

**什么问法：** "开场那个变形球是怎么做的？""OBJ 序列你怎么生成和播放的？"

**怎么答：**

分三步。第一步生成——程序启动时检查 `media/models/seq/` 下有没有帧文件，没有就自己生成：用 UV 球面的参数化（segments × rings），每帧按 `phase = frameIndex / totalFrames` 算半径——主体脉冲 `0.25*sin(phase*2π)` 加上纵向波纹 `0.14*sin(4φ + phase*6)`——然后用自研的 `saveObj()` 写到磁盘。第二步播放——`ObjSequence` 类把 30 帧全 load 成独立的 `Model` 对象，计时器每 `1/fps` 秒切到下一帧，硬件上每帧是独立的 VBO/EBO。第三步是真正的多帧连续绘制——每一帧读不同的 .obj，网格顶点位置不同，不是平移缩放。这就是基本要求第七项。

## Q2：为什么不直接在内存里生成就好，还要写文件再读？

**什么问法：** "你生成到磁盘又读回来，这样绕一圈有什么意义？""你在内存里直接生成网格不行吗？"

**怎么答：**

因为课程要求第七条写的就是"读取连续的 .obj 文件"——文件必须在磁盘上，读回来才算。我们选择了自产自销的方案：用自研的 `writeObj` / `saveObj` 生成，用自研的 `loadObj` / `Model(filepath)` 读取播放。好处是验证了 OBJ 导出→加载的完整闭环，不依赖外部工具。如果是在内存里直接 swap 网格，课程要求这条就没法体现。

## Q3：miniaudio 怎么集成的？为什么不自己写音频？

**什么问法：** "音频部分怎么做的？""miniaudio 是什么东西？"

**怎么答：**

miniaudio 是单头文件的跨平台音频库，CC0 协议。只有一个 C 头文件，不需要链接额外的 .lib 或 .dll。我们用它的 `ma_engine` 高级 API——初始化引擎后，`ma_sound_init_from_file` 加载 wav 文件，`ma_sound_start` 播放。射击和命中时调用 `play("pistol")`、`play("sniper")`、`play("pop")`。

关于为什么不自己写——音频解码和平台后端（Windows 走 WASAPI、Linux 走 ALSA/PulseAudio）是操作系统/信号处理问题，不是图形学课程范围。课程不要求自研音频，所以选了一个轻量第三方库。OBJ、NURBS、阴影、碰撞这些图形学核心都自研了，音频没必要重复造轮子。

## Q4：音效文件哪来的？是 CC0 的？

**什么问法：** "这三个 wav 是什么来源？""音效是不是网上下的？"

**怎么答：**

pistol、sniper、pop 三个 wav 是我们自己合成出来的，不是下载的。每个音效用一个生成器小程序——写 WAV header（RIFF chunk + fmt chunk + data chunk，16-bit PCM 单声道 22050Hz），然后用正弦波加白噪声合成采样。手枪是 700Hz 正弦加噪声，极短衰减（~0.1s），模拟消音器闷响。狙击是 130Hz 低频 boom 加缓慢衰减（~0.28s），模拟重击感。pop 是 1100Hz 向下扫频的 ping（~0.15s），模拟气球破裂。全部是程序生成的。这就是上午你问的那个狙击枪声没声音的 bug——之前加载在 mp3 路径，mingw 的 miniaudio 最后没编进 MP3 解码器支持，我把加载路径切到 wav 就解决了。

## Q5：miniaudio 的 MP3 支持为什么有问题？怎么解决的？

**什么问法：** "你刚才说 mp3 切到 wav，为什么 mp3 不行？"

**怎么答：**

miniaudio 内置了 MP3 解码器，但编译时需要 `#define MINIAUDIO_IMPLEMENTATION` 并且不定义 `MA_NO_MP3`。我们的 `audio.cpp` 里确实定义了 `MINIAUDIO_IMPLEMENTATION` 且没禁用 MP3。但 miniaudio 的 MP3 支持是可选特性——有些工具链/版本中默认就是关的，或者需要编译器宏显式开启 `MA_ENABLE_MP3`。我们的 mingw 环境下加载 mp3 时 `ma_sound_init_from_file` 返回失败，解码器没有内置进去。问题很简单——换回 wav 就行了。这个可以进一步确认 `MINIAUDIO_IMPLEMENTATION` 定义位置，以及 miniaudio 的 mp3 解码宏。

## Q6：开场动画里那个球能被打中，怎么做的射线检测？

**什么问法：** "你说进场动画还能和球互动，怎么做的？"

**怎么答：**

开场动画的球同样用 ray-sphere 解析判别式检测——`tryFire()` 里对每个活靶子算 `b = dot(oc, dir), c = |oc|² - r², disc = b² - c`，disc 非负且取最近正根就是命中。命中后先给粒子爆炸反馈（蓝色三层），然后计时器 1.2 秒后走 `_intro = false` 结束动画进游戏。这个球和后面的靶子共用同一套命中检测。

## Q7：你负责的 OBJ 序列 + 音频，加起来也就几百行代码，工作量怎么体现？

**什么问法：** "你这部分写了多少代码？""你的贡献和另外两位相比，怎么量化？"

**怎么答：**

先说清楚我的三块活：
- **ObjSequence 类**——生成器（UV 球面变形、flatten 到顶点数组、`saveObj` 写文件）、播放器（按帧率 clock 切 Model、重绕、loop 控制）。约 130 行。
- **Audio 封装**——`audio.cpp`（miniaudio `ma_engine` 初始化 + `ma_sound` 管理 + play/stop 封装），约 70 行。
- **aim_trainer 的集成**——intro 状态机（`_intro` 判定 + skip 规则 + `renderIntro` 固定摄影机路径 + `drawIntroOverlay` 标题过渡 + `captureScreenshot` F2 路径 + `exportSceneObj` 合并导出），以及手电、ESC 暂停这些 QoL。这些约 180 行。

总共大约 380 行新代码。但我的另一块内核工作是调试和工具链适配——MinGW 下 cmake 因为 Git 的 sh.exe 报编译器 broken，我在 settings.json 里把任务切到干净的 PATH 才跑通，这个能跑的环境本身就是一个工程成果。所以不能只看行数。

## Q8：调试答辩用的 pre-built exe 能跑通吗？

**什么问法：** "演示前确认能跑吗？""在另一个机器上会出问题吗？"

**怎么答：**

会的——每一轮新编译我会要求 `cmake --build build` 的 exit code = 0 才算通过。构建环境和运行环境严格解耦：POST_BUILD 把 `media/` 目录全量拷贝到 exe 同级，所以不用依赖源码树中的路径。只要构建机器上 g++ / cmake / mingw32-make 在 PATH 里即可。答辩用的那台电脑上会先重新 `cmake -B build && cmake --build build` 一编。注意：PowerShell 下因为 Git 的 `sh.exe` 在 PATH 里，`cmake -G "MinGW Makefiles"` 会崩，必须用设置好的 VSCode 任务（带干净 PATH）来跑。

## Q9：COX-DE BOOR 原理简述

**什么问法：** "能简单说一下 Cox-de Boor 递推吗？""NURBS 的基函数逻辑？"

**怎么答：**

对——虽然 NURBS 是柳舟主负责的，但如果助教针对"自研 NURBS"提问，我会说：

Cox-de Boor 是 B 样条基函数的标准递推定义。P 等于零的底 case：参数落在当前节点跨段内返回 1，否则 0。P 大于零的递推 case：节点差值的线性组合——左半边是 `(t - t_i) / (t_{i+p} - t_i) * N_{i,p-1}(t)`，右半边是 `(t_{i+p+1} - t) / (t_{i+p+1} - t_{i+1}) * N_{i+1,p-1}(t)`。我们的实现里分母如果接近于 0 会跳过该支，不会崩。整体复杂度 O(p²)。

## 不要被问到但不知道怎么答的细节

| 可能追问 | 关键词 |
|------|------|
| OBJ 帧的 vertex count | UV sphere：28 segments × 18 rings ≈ 550 verts/帧 |
| 生成 fps | 12 fps，30 帧 = 约 2.5 秒一次循环 |
| `saveObj` 格式 | 简化的 p/t/n 一一对应，`f i/i/i` 格式，不加 mtl 材质引用 |
| `captureScreenshot` | `glReadPixels(GL_RGBA, GL_UNSIGNED_BYTE)` → 翻转 Y → `stbi_write_png` |
| `exportSceneObj` | 合并障碍物 AABB + 平台 + 斜坡 + 靶子 → 世界坐标变换后一起 `saveObj` |
| miniaudio 版本 | 单头文件，精确版本号在 `miniaudio.h` 头部，当前使用的是 `master` 分支 snapshot |
| 为什么不用 `ma_sound_group` | 因为一个 group 的 play 会打断上一个声音——我们想连续射击每次播放独立的声音实例，而不是被前一次打断。所以每个名字只注册了一个 `ma_sound`，射击时 seek 到 0 然后 re-play |
